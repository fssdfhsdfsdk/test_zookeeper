  单个服务能感知所有服务 以及 变化
有价值、实际工业的场景：分布式块存储 + zk
一个**完整闭环的工业级分布式存储系统**
- **ZK 集群**：仲裁者（3 节点）。
- **OSD (Object Storage Daemon)**：存储节点（3 节点），负责存实际数据，向 ZK 注册容量。
- **MDS (Metadata Server)**：元数据管理节点（2 节点，主备模式），通过 ZK 选举**唯一 Active 主节点**，
添加一个client客户端，通过MDS获取osd信息以及其变化，并通过一致性hash将block均匀分配给osd。并模拟写入数据，每次写入一个block（通过key-value模拟），读取block数据。通过文件模拟底层磁盘，每个osd管理一个磁盘。
 client可以创建多个块设备，每个块设备由多个block组成
 client支持cli进行读取、写入某个盘
1. **数据副本 (Replication)**：写入时，顺时针写 N 个节点（如 3 个）。读取时读最新版本。
2. **版本向量 (Vector Clock)**：解决并发写入冲突。
3. **后台数据迁移 (Rebalancing)**：当新 OSD 加入，旧 OSD 需要主动将属于新 OSD 哈希范围的数据迁移过去（本教程是写入时自动路由到新节点，旧数据仍在旧节点，需要后台进程迁移）。
